### User Story:
**Trigger**: Rapidly degrading upload durability (spike in 1-copy/0-copy outcomes).
**Need**: Alert within minutes for on-call to investigate specific SPs.
**Goal**: Avoid delayed incident discovery from weekly trend analysis.

### Context:  
Fast detection over precision. Single noisy ticks are acceptable; sustained drops across multiple ticks require alerts.

### Acceptance Criteria:
- **Granularity**: Metrics computed per reactive tick (5–15 min ideal).  
- **Alerting**: Fires when the reactive window drops below threshold for N consecutive ticks.  
- **Routing**: Alerts sent to standard telemetry ops channel.  
- **Payload**: Include % drop, absolute counts, impacted SPs, dashboard link.  
- **Environment**: Thresholds are environment-specific (e.g., mainnet vs calibration).

### Dependencies and Considerations:
- **Log Cadence**: Real-time SP log delivery required; longer cadences (e.g., hourly) limit detection speed. Highlight upstream prerequisites if needed.  
- **Validation**: Verify in “windows” feature; escalate to Curio maintainers for unresolved issues.
