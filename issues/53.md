### User Story
Telemetry should proactively monitor and detect retrieval failures per dataset/SP. This ensures failures are identified and diagnosed internally before customers report issues.

### Acceptance Criteria:
- Track and filter retrieval outcomes by dataset ID.
- For each dataset, indicate which SP handled the read and whether it succeeded/failed.
- Ensure data is sortable by recency.

### Dependencies:
- Requires dataset IDs in relevant events ([#41](https://github.com/FilOzone/sp-telemetry-stories/issues/41)).
